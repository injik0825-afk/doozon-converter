# 더존 위하고 전표 변환기 — 사용 가이드

거래내역 엑셀 파일을 더존 위하고 일반전표 업로드 형식으로 자동 변환하는 Streamlit 앱입니다.

---

## 📦 지원 회사

| 회사 | 자동 감지 키워드 | 계정코드 |
|---|---|---|
| 리버사이드파트너스 | `IBK 기업은행`, `한국투자증권`, `교보증권`, `비씨카드` 등 | 5자리 (10301, 11100…) |
| 두리인베스트먼트 | `IBK기업은행 내역`, `키움(…)`, `메리츠(…)`, `한국(…)`, `교보(…)` 등 | 3자리 (103, 107…) |

업로드된 파일의 시트명을 보고 회사를 자동으로 판단합니다.

---

## 🚀 매월 사용 방법

1. 거래내역 엑셀 파일들을 다운로드해서 한 번에 업로드 (여러 파일 동시 가능)
2. **🔄 변환 시작** 버튼 클릭
3. 화면에 아래 정보 표시됨:
   - 🏢 자동 감지된 회사명
   - 📊 전표 행 수, 미분류 건수, 차대변 합계
   - 🆕 새 종목 감지 시 추가 코드
4. **📥 변환 파일 다운로드** 클릭하면 색상이 적용된 엑셀 파일 받음
5. 더존 위하고에 업로드하기 전, **🔴 빨간 행은 직접 수정** (취득가 등)

---

## 🎨 변환 결과 색상 의미

| 색상 | 의미 | 조치 |
|---|---|---|
| 🔴 빨간 행 | 취득가 확인 필요 (금액 0) | 매수 단가 직접 입력 |
| 🟠 주황 행 | 미분류 거래 | 수동 확인 후 계정 입력 |
| 🟡 노란 행 | 헤더 | - |

## 🆕 새 종목 추가 방법

매월 새로운 주식/채권에 투자할 때 변환 후 화면 하단에 **"🆕 신규 종목 감지"** 박스가 나타나면, 그 안의 코드를 복사해서 Claude에게 보내주세요. 시장 구분(코스피/코스닥/회사채 등)만 알려주시면 바로 코드에 반영해드립니다.

예시:
```python
# STOCK_DB 추가분:
    '새종목A': '주식#코스닥#새종목A',  # ← 코스피/코스닥/회사채 확인 필요
```

## ⚠️ 자동변환 영역 외 (수동 입력 필요)

거래내역 파일에 원천 데이터가 없는 **결산성 분개**는 자동변환되지 않습니다. 회계사가 결산 시점에 직접 입력하셔야 해요.

### 두 회사 공통

- 월말 평가상계 / 평가이익 / 평가손실
- 손익계정 대체 / 이월이익잉여금 / 미처분이익잉여금

### 리버사이드 (3월 기준)

- 직원/임원급여 결산 분개 (`급여등지급` 전표 4M)
- 미지급비용 반제 (전월말 미지급분 3.28M)
- 티엠씨 매도 3건 → 빨간 행에 매수 단가(@18,620) 직접 입력

### 두리인베스트먼트 (3월 기준)

- 직원/임원 급여 (12.5M), 상여금 (120M), 퇴직급여 (281M)
- 법인세등 (105.9M), 미지급세금 (28M)
- 미처분/이월이익잉여금/손익 결산 (76억)
- 신민자 퇴직금 중간정산 (16.4M) - 미분류로 표시되니 직접 분개

### 리버사이드파트너스
- ✅ **11개 계정 완전 일치** (보통예금, 선급금, 단매증 차변, 미수금, 미지급세금, 보험료(판), 복리후생비(판), 선납세금, 세금과공과금(판), 접대비(기업업무추진비)(판), 주식거래수수료(판), 지급수수료(판))
- 🔴 빨간 행 3건 (티엠씨 매도)
- 미분류 0건

### 두리인베스트먼트
- ✅ **3개 계정 완전 일치** (접대비(기업업무추진비)(판), 지급임차료(판), 보험료(판))
- 🟡 **3개 계정 5% 이내 근접** (복리후생비(판) 3.3%, 여비교통비(판) 4.8%, 이자수익(금융) 0.02%)

- 🎯 **미지급금 차변 거의 일치** (정답 112.8M / 변환 109.6M)
- 미분류 51건 (대부분 결산성 분개로 자동변환 영역 외)

## 🔄 GitHub 업로드 방법

1. GitHub `doozon-converter` 저장소 → `app/streamlit_app.py` 클릭
2. 우측 상단 `...` → **Delete file** → Commit
3. `app/` 폴더로 돌아가서 **Add file → Upload files**
4. `streamlit_app.py` 파일을 드래그 앤 드롭

6. 1~2분 후 Streamlit Cloud가 자동 재배포

## 🛠️ 기술 정보

- **언어**: Python 3.10+ (Streamlit)
- **주요 라이브러리**: pandas, openpyxl
- **회사 감지**: 시트명 패턴 매칭 (정규식)
- **변환 로직**:
  - 한국증권/교보/IBK은행 (리버사이드) → 기존 파서
  - 한국증권/교보/키움/메리츠/IBK기업은행/카드이용내역 (두리) → 두리 전용 파서

### 중복 방지

- O하이/O채권 22건: IBK 입금은 무시 (교보 시트의 `은행이체출금`에서 분개)
- 두리 HTS당사이체/HTS타사이체: 한국증권/교보/키움 시트에서 무시 (IBK에서 분개)
- 두리 은행거래내역/납입내역 시트: IBK와 중복이라 무시

## ❓ 문제 해결

### "변환된 데이터가 없습니다" 오류
- 시트명이 인식 가능한 패턴인지 확인
- 빈 파일 또는 형식 깨진 파일 가능성

### 자동 감지가 안 됨
- 시트명에 회사 식별 키워드가 없는 경우 → 리버사이드로 처리됨
- Claude에게 시트명 알려주면 감지 패턴 추가 가능

### 변환 결과가 분개장과 차이남
- 결산성 분개는 자동변환 안 됨 (위 ⚠️ 영역 참고)
- 회사 회계 정책이 달라진 경우 Claude에게 분개장 보내서 패턴 업데이트 요청

### 새 종목/거래처 추가, 분류 규칙 변경
→ Claude에게 요청해주세요
