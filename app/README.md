## 🚀 사용 방법

1. 거래내역 엑셀 파일들 다운로드 → Streamlit 앱에 업로드 (여러 파일 동시 가능)
2. **🔄 변환 시작** 버튼 클릭
3. 화면 표시:
   - 🏢 자동 감지된 회사명
   - 📊 전표 행 수, 미분류 건수, 차/대변 합계
   - 🆕 새 종목 감지 시 추가 코드
4. **📥 변환 파일 다운로드** → 색상 적용된 엑셀 파일 받기
5. 더존 위하고 업로드 전 **🔴 빨간 행은 직접 수정** (취득가 등)

**증권 거래 통합 로직 (`_central_sec_row`) 재작성 — 신고된 오류 전부 수정**
- 거래처 표기 정정: 예치금/배당/이자측 `증권사(계좌유형)`, 주식/수수료/세금측 `주식#시장#종목`. (기존 `(센트럴밸류-유형)` 형식 제거). iM→`IM증권`, DB 고유→`DB금융투자(고유)`, 한화→`한화증권(유형)`.
- 적요에 단가 포함: `종목(수량주*@단가)매수/매도/입고/납입`.
- **공모주 청약·입고 취득원가 = 수량×단가×1.01** (청약 1% 수수료). 분개장 5건(키움히어로 1,123,120 / 신한제18호 1,018,080 / 채비 285,729 / 코스모로보틱스 6,720,540 / 교보20호 808,000) 전수 일치.
- 매도: 예치금(정산) + 거래수수료(828) + 세금과공과금(817) 생성, 단매증 대변은 취득원가 미상이므로 🔴RED.
- 예탁금이용료/정기이자 → 이자수익(금융) 420 (기존 기타제예금/이자수익 오류 수정). 배당/주식합병입금/분배금 → 배당금수익 419.
- 은행이체·HTS타사이체 입출금 → 예치금 ↔ 보통예금(🟠ORANGE). 동일 증권사 대체 입출금 → 예치금 ↔ 예치금(🟠ORANGE).
- 자금부입금법인청약 제외(사용자 요청). 순수 보관이전(타사/이체/대체 입·출고, 합병 입·출고) 제외.

**파서/판독 버그 수정**
- nh_sec·kb·hantoo·kiwoom: 신규 13인자 시그니처로 호출(기존 인자 누락으로 변환 실패하던 문제). KB 정산 col4·거래세 col5, NH 세금 col8, 한투 거래세 r1 col8.
- **키움 묶음매도**: 정산금액이 묶음 마지막 행에 누적 표기되어 이중계상 → 건별 거래금액으로 net 계산(settle=0). 거래세(col4)는 마지막 행 총액 1회만 반영해 분개장과 정확히 일치(예치금 3,052,063·증권거래세 6,117·수수료 450).
- **DB금융투자**: 2행/거래 + 날짜가 A행 col1·종목명 B행 col2인 실제 구조로 재작성(기존 입출금형 컬럼 가정 오류).
- 하나/한화/신영/메리츠: 병합 2줄 헤더 → 데이터 시작 `hdr+2` 보정(기존 hdr+1로 미파싱).
